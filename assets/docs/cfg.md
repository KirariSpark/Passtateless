本文档说明了自定义生成器所支持的所有操作命令（`name`）、对应的参数列表（`args`）及其默认值。

## 命令格式

每个命令都是字典，包含以下字段：
- `name` (String, **必填**)：要执行的操作名称。
- `args` (List<dynamic>, **可选**)：操作所需的参数列表。如果未提供或为 `null`，将默认当作空列表 `[]` 处理。
- 参数部分使用位置参数。

## 变量参数

如果参数列表中的某个值为字符串且以 `#` 开头，解析器将其视为**变量引用**，会在执行命令前自动替换为变量的当前值。

### 支持的变量
| 变量名         | 说明                  |
|:------------|:--------------------|
| `#password` | 代表当前步骤已生成的半成品密码字符串。 |

**示例：**

```json
[
  { "name": "append", "args": ["#password"] }
]
```

此配置会将当前的密码内容复制并追加到自身末尾（即密码内容翻倍）。

## 可用操作列表

### 编码与哈希

| 操作键         | 说明                              | 参数 (`args`)                                | 默认值                                   |
|:------------|:--------------------------------|:-------------------------------------------|:--------------------------------------|
| `toBase64 ` | 将当前密码转换为 Base64 编码              | 无                                          | -                                     |
| `toSHA256`  | 将当前密码转换为 SHA256 哈希值             | 无                                          | -                                     |
| `toPBKDF2`  | 使用 PBKDF2 算法（基于Hmac-SHA256）进行哈希 | `[0]` salt (String) `[1]` iterations (int) | `iterations`: `100000` *(注：salt 为必填)* |

### 字符过滤与清理

| 操作键            | 说明                       | 参数 (`args`) | 默认值 |
|:---------------|:-------------------------|:------------|:----|
| `removeSpChar` | 移除密码中的所有特殊字符             | 无           | -   |
| `removeAlpha`  | 移除密码中的所有字母               | 无           | -   |
| `removeDigits` | 移除密码中的所有数字               | 无           | -   |
| `deduplicate`  | 去除密码中连续或重复的字符，仅保留首次出现的字符 | 无           | -   |

### 字符串变换

| 操作键              | 说明              | 参数 (`args`)                                | 默认值                                   |
|:-----------------|:----------------|:-------------------------------------------|:--------------------------------------|
| `reverse`        | 完全反转整个密码字符串     | 无                                          | -                                     |
| `reversePartial` | 反转密码中指定区间的字符    | `[0]` start (int) `[1]` end (int)          | 无 *(注：start 和 end 均为必填)*              |
| `rotate`         | 旋转密码字符（左移或右移）   | `[0]` steps (int) `[1]` direction (String) | `direction`: `"left"` *(注：steps 为必填)* |
| `extract`        | 从密码中间隔抽取字符组成新密码 | `[0]` step (int) `[1]` start (int)         | `step`: `1` `start`: `0`              |
| `repeat`         | 将密码重复拼接指定的次数    | `[0]` times (int)                          | `times`: `1`                          |

### 长度控制与填充

| 操作键         | 说明                    | 参数 (`args`)                                                    | 默认值                                              |
|:------------|:----------------------|:---------------------------------------------------------------|:-------------------------------------------------|
| `crop`      | 截取密码的前 N 个字符          | `[0]` length (int)                                             | `length`: `16`                                   |
| `pad`       | 在密码的开头或结尾填充指定字符至目标长度  | `[0]` length (int) `[1]` char (String) `[2]` position (String) | `length`: `16` `char`: `"A"` `position`: `"end"` |
| `setLength` | 强制设定密码长度（过长则截断，过短则填充） | `[0]` length (int) `[1]` char (String) `[2]` position (String) | `length`: `16` `char`: `"A"` `position`: `"end"` |

### 插入与追加

| 操作键      | 说明         | 参数 (`args`)                                  | 默认值                                |
|:---------|:-----------|:---------------------------------------------|:-----------------------------------|
| `insert` | 在指定位置插入字符串 | `[0]` inserted (String) `[1]` location (int) | `location`: `0` *(注：inserted 为必填)* |
| `append` | 在密码末尾追加字符串 | `[0]` appended (String)                      | 无 *(注：appended 为必填)*               |

### 随机插入操作

*(注：以下操作的随机性基于密码当前状态的 SHA256 前 7 位 ASCII 码之和作为基础种子)*

| 操作键               | 说明             | 参数 (`args`)                             | 默认值                            |
|:------------------|:---------------|:----------------------------------------|:-------------------------------|
| `insertRandDigit` | 随机插入数字字符 (0-9) | `[0]` amount (int) `[1]` seed (int)     | `amount`: `1` `seed`: `0`      |
| `insertRandAlpha` | 随机插入字母字符       | `[0]` amount (int) `[1]` seed (int)     | `amount`: `1` `seed`: `0`      |
| `insertRandSp`    | 随机插入特殊字符       | `[0]` amount (int) `[1]` seed (int)     | `amount`: `1` `seed`: `0`      |
| `shuffle`         | 随机打乱密码中的字符顺序   | `[0]` iterations (int) `[1]` seed (int) | `iterations`: `10` `seed`: `0` |

## 错误处理说明

如果在执行过程中发生以下情况，解析器将中止后续命令并返回相应的错误码（`ErrorCode`）：
* **`unknownCommand`**：传入的 `name` 不在上表列出的操作键中。
* **`invalidArgs`**：
    * 参数类型转换失败（如需要 `int` 但传入了 `String`）。
    * 缺少必填参数导致索引越界。
    * 变量解析错误（如使用了不支持的变量名）。
    * `Generator` 内部校验不通过（如 `step` 小于等于 0、插入空字符串或包含空格的字符串等）。

## 示例

假设输入的原始字符串包含中文或特殊 Unicode 字符（例如：`"我的密码P@ss!包含中文"`），为了生成只包含字母和数字的标准化密码，可以先进行 Base64 编码将其转为纯 ASCII 字符，再移除 Base64 特有的符号（如 `+`、`/`、`=`），最后截取所需的长度。

**JSON 配置如下：**
```json
[
  { "name": "toBase64" },
  { "name": "removeSpChar" },
  { "name": "crop", "args": [16] }
]
```

**执行流程解析：**

1. **`toBase64`**：将包含 Unicode 的原始输入转换为标准的 Base64 字符串。
2. **`removeSpChar`**：移除上一步产生的所有特殊字符，确保结果仅剩字母和数字。
3. **`crop`**（后续处理）：将纯字母数字字符串截取为固定长度（此处设为 16 位），得到最终的可用密码。

**注意：** 这个示例实际上相当弱，因为操作太少，实际配置应比它复杂得多。
