## 基本结构
JSON 的核心结构只有两种：
- 对象：使用花括号 `{}` 表示，包含一系列无序的“键值对”。
- 数组：使用方括号 `[]` 表示，包含一系列有序的值。  

键值对的书写格式为：
```json
"key": value
```

其中键必须是字符串，且必须用英文双引号 `""` 包裹。

## 支持的数据类型

JSON 支持 6 种基本数据类型：

| 类型  | 描述                         | 示例                      |
|:----|:---------------------------|:------------------------|
| 字符串 | 用双引号包裹的文本，支持转义字符           | `"Hello World"`         |
| 数字  | 整数或浮点数，不能有前导零（如 `01` 是非法的） | `42`, `3.14`, `-1.5e10` |
| 布尔值 | 逻辑值，全部小写                   | `true`, `false`         |
| 空值  | 表示空值，全部小写                  | `null`                  |
| 对象  | 无序的键值对集合，嵌套在 `{}` 中        | `{"name": "Alice"}`     |
| 数组  | 有序的值列表，可以包含不同类型，嵌套在 `[]` 中 | `[1, "two", true]`      |

## 语法规则

- 无论是对象的键，还是字符串类型的值，都只能使用双引号 `""`，单引号 `''` 或无引号都会导致语法错误。
- 对象的最后一个键值对，或者数组的最后一个元素后面，不能加逗号。
- 标准的 JSON 文件中不允许添加注释。如果需要注释，通常只能在包裹 JSON 的代码行中注释。
- 一个合法的 JSON 文本，它的根元素必须是一个对象 `{}` 或是一个数组 `[]`，不能只是一个孤立的字符串或数字。

## 示例

### 错误的 JSON
```json
{
  name: "张三",           // 键没有加双引号
  'age': 25,             // 键使用了单引号
  "isStudent": True,     // 布尔值首字母大写了
  "hobbies": ["reading", "swimming",], // 数组和键值对末尾有多余的逗号
  // 这个注释本身也是一个错误
}
```

### 正确的 JSON
```json
{
  "name": "张三",
  "age": 25,
  "isStudent": true,
  "hobbies": [
    "reading",
    "swimming"
  ]
}
```
