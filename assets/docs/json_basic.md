**JSON**（JavaScript Object Notation）是一种轻量级的**数据交换格式**。它基于 JavaScript 的对象字面量语法，但与语言无关，现已成为跨平台、跨语言传输和存储数据的标准格式。
JSON 具有易于人类阅读和编写，同时也易于机器解析和生成的特点。

## 1. 基本结构
JSON 的核心结构只有两种：
*   **对象**：使用花括号 `{}` 表示，包含一系列无序的“键值对”。
*   **数组**：使用方括号 `[]` 表示，包含一系列有序的值。
    键值对的书写格式为：
```json
"key": value
```
**注意：** 键必须是**字符串**，且必须用**双引号** `""` 包裹。

## 2. 支持的数据类型
JSON 支持 6 种基本数据类型：
| 类型 | 描述 | 示例 |
| :--- | :--- | :--- |
| **字符串** | 用双引号包裹的文本，支持转义字符（如 `\n`, `\"`） | `"Hello World"` |
| **数字** | 整数或浮点数，不能有前导零（如 `01` 是非法的） | `42`, `3.14`, `-1.5e10` |
| **布尔值** | 逻辑值，全部小写 | `true`, `false` |
| **空值** | 表示空值，全部小写 | `null` |
| **对象** | 无序的键值对集合，嵌套在 `{}` 中 | `{"name": "Alice"}` |
| **数组** | 有序的值列表，可以包含不同类型，嵌套在 `[]` 中 | `[1, "two", true]` |

## 3. 严格的语法规则
与一些宽松的编程语言不同，JSON 的语法**非常严格**，常见的易错点如下：
1.  **必须使用双引号**：无论是对象的键，还是字符串类型的值，都**只能**使用双引号 `""`，单引号 `''` 或无引号都会导致语法错误。
2.  **不能有尾随逗号**：对象的最后一个键值对，或者数组的最后一个元素后面，**不能**加逗号。
3.  **不支持注释**：标准的 JSON 文件中不允许添加注释（如 `//` 或 `/* */`）。如果需要注释，通常只能在包裹 JSON 的代码行中注释。
4.  **最外层必须是对象或数组**：一个合法的 JSON 文本，它的根元素必须是一个对象 `{}` 或是一个数组 `[]`，不能只是一个孤立的字符串或数字。

## 4. 合法与非法示例对比
### ❌ 非法的 JSON
```json
{
  name: "张三",           // 错误：键没有加双引号
  'age': 25,             // 错误：键使用了单引号
  "isStudent": True,     // 错误：布尔值首字母大写了
  "hobbies": ["reading", "swimming",], // 错误：数组末尾有多余的逗号
  "address": undefined   // 错误：JSON 不支持 undefined
}
```
### ✅ 合法的 JSON
```json
{
  "name": "张三",
  "age": 25,
  "isStudent": true,
  "hobbies": [
    "reading",
    "swimming"
  ],
  "address": null
}
```
